Bilgisayar Bilimi, Veri Yapıları ve Algoritmalar

Sadece kod yazan değil, yazdığı kodun maliyetini ve performansını hesaplayan bir mühendis olmanın temelleri.

📊 Temel Veri Yapıları (Data Structures)

Veriyi bellekte nasıl tuttuğumuz, algoritmanın hızını belirler:

  • Arrays (Diziler): Bellekte yan yana sıralı kutulardır. İndis (index) ile veriye erişim çok hızlıdır (O(1)) ama araya eleman eklemek maliyetlidir.
  • Objects / Hash Maps: Anahtar-değer (Key-Value) çiftleriyle çalışır. Benzersiz bir anahtarla arama yaparken veriyi anında bulur.
  • Stacks & Queues: Stack mantığı "Son Gelen İlk Çıkar" (LIFO - Örn: Tarayıcı geri butonu). Queue mantığı "İlk Gelen İlk Çıkar" (FIFO - Örn: Sunucu istek kuyruğu).

⏱️ Zaman Karmaşıklığı (Big O Notation)

Yazdığın algoritmanın veri miktarı arttıkça ne kadar yavaşlayacağını ölçme standardıdır:

  • O(1) - Sabit Zaman: Veri boyutu ne olursa olsun işlem süresi değişmez. (Örn: Dizinin ilk elemanını okumak).
  • O(n) - Lineer Zaman: Veri arttıkça süre de doğru orantılı artırır. (Örn: Dizide döngüyle bir eleman aramak).
  • O(n²) - Karesel Zaman: İç içe iki döngü demektir, büyük verilerde sistemi kilitler. Kaçınılması gerekir.

Gelişmiş Algoritma Analizi: Bubble Sort (Kabarcık Sıralaması) Mantığı

Dizideki yan yana elemanları karşılaştırıp yerlerini değiştirerek yapılan basit ama yavaş (O(n²)) bir sıralama algoritması mantık akışı:

FONKSİYON bubbleSort(dizi) {
    let n = dizi.uzunluk;
    FOR (let i = 0; i < n; i++) {
        FOR (let j = 0; j < n - i - 1; j++) {
            EĞER dizi[j] > dizi[j+1] İSE {
                // Elemanların yerini takas et (Swap)
                let gecici = dizi[j];
                dizi[j] = dizi[j+1];
                dizi[j+1] = gecici;
            }
        }
    }
    DÖNDÜR dizi;
}

Kurumsal HTML5 Mimarisi & SEO Standartları

Arama motorlarında üst sıralara çıkan, ekran okuyuculara uyumlu ve modern multimedya destekli yapılar.

Veri Depolama Girişi: Web Storage API

HTML5 ile birlikte çerezler (cookies) yerine tarayıcı hafızasında veri saklamak için iki devasa mekanizma geldi:

  • LocalStorage: Kullanıcı tarayıcıyı, hatta bilgisayarı kapatsa bile veriler silinmez (Örn: Tema tercihi, sepet bilgisi).
  • SessionStorage: Veriler sadece açık olan sekme kapatılana kadar yaşar, sekme kapandığı an imha edilir.

İleri Mimari CSS, Grid System & Duyarlılık

Karmaşık iki boyutlu dashboard düzenleri, CSS Değişkenleri ve Mobile-First tasarım yaklaşımları.

🏁 İki Boyutlu CSS Grid Düzeni

Flexbox tek yönlü (satır veya sütun) hizalama yaparken, Grid sistemi aynı anda hem satırı hem sütunu tek merkezden yönetir:

.container {
    display: grid;
    grid-template-columns: repeat(3, 1fr); /* 3 eşit sütun */
    grid-template-rows: auto;
    gap: 20px; /* Elemanlar arası boşluk */
}

📱 Medya Sorguları (Media Queries)

Sitenin telefon, tablet ve monitör ekranlarına göre otomatik şekil değiştirmesini sağlar (Responsive):

/* Mobil öncelikli tasarım bittikten sonra tablet/masaüstü ayarı */
@media (min-width: 768px) {
    .sidebar { display: block; }
    .main-content { font-size: 16px; }
}

🎨 CSS Değişkenleri (Custom Properties) & Animasyonlar

Kod tekrarını önlemek için global renk ve boyut değişkenleri tanımlayıp, `@keyframes` ile donanım hızlandırmalı akıcı animasyonlar tetikleyebilirsiniz:

:root {
    --ana-renk: #4f46e5;
}
.buton { background-color: var(--ana-renk); }

@keyframes nabiz {
    0% { transform: scale(1); }
    50% { transform: scale(1.05); }
    100% { transform: scale(1); }
}

Modern JavaScript (ES6+) & Asenkron Mimari

Dizileri uçurma yöntemleri, yıkıcı atamalar ve uzak sunuculardan (API) veri çekme mekanizmaları.

🚀 Modern ES6+ Standartları

Kod yazma hızını ve okunabilirliğini artıran yeni nesil JavaScript sözdizimi:

// Arrow Function (Ok Fonksiyonu)
const topla = (a, b) => a + b;

// Destructuring (Objeyi/Diziyi Parçalama)
const user = { ad: "Can", rol: "Admin" };
const { ad, rol } = user; // Tek hamlede dışarı çıkarma

🔄 İleri Dizi Metotları

Klasik `for` döngüleri yerine dizileri manipüle eden işlevsel fonksiyonlar:

const sayilar = [1, 2, 3, 4];
// Tüm elemanları 2 ile çarpıp yeni dizi üretir:
const katlar = sayilar.map(x => x * 2); 
// 2'den büyük olanları filtreler:
const buyukler = sayilar.filter(x => x > 2);

🌐 Asenkron JS: Promises, Async/Await & Fetch API

JavaScript tek kanallı (single-threaded) bir dildir. Uzak bir sunucudan veri çekilirken uygulamanın donmaması için asenkron yapıları kullanırız:

// Gerçek zamanlı dış dünyadan API verisi çekme fonksiyon kalıbı
async function verileriGetir() {
    try {
        let response = await fetch('https://api.github.com/users/github');
        let data = await response.json(); // Gelen ham metni JSON objesine çevirir
        console.log("Kullanıcı Adı: " + data.name);
    } catch (hata) {
        console.error("Veri alınırken hata oluştu kanka: ", hata);
    }
}

Veritabanı Mimarileri & Git Sürüm Kontrolü

Büyük verileri nerede saklayacağımızı seçmek ve projeleri ekip halinde hatasız yönetmek.

🗄️ SQL vs NoSQL Karşılaştırması

Veritabanı türünü projenin ihtiyacına göre seçmeliyiz:

  • Relational (SQL): Veriler tablolar, satırlar ve katı kurallarla tutulur. İlişkisel yapılardır (Örn: MySQL, PostgreSQL). Finansal işlemler ve e-ticaret için harikadır.
  • Non-Relational (NoSQL): Veriler esnek JSON dokümanları şeklinde tutulur (Örn: MongoDB). Çok hızlı genişler (Scalability) ve anlık büyük veri akışlarında (chat, loglama) tercih edilir.

🌿 Temel Git Komut Kütüphanesi

Kodların sürümlerini kaydetmek ve GitHub/GitLab üzerine yüklemek için kullanılan çekirdek komutlar:

  • git init : Bulunduğun klasörde boş bir Git deposu başlatır.
  • git add . : Değişen tüm dosyaları kayıt listesine ekler.
  • git commit -m "mesaj" : Değişiklikleri yerel hafızaya mühürler.
  • git push origin main : Kodları buluttaki ana sunucuya postalar.
